Chapter 1: DATABASE

With this chapter, we are going to explain MySQL commands used in our system to build tables, create relationships between tables, and manipulate data as we can view, insert, update, and delete.

In our database, we need to explain table-to-table by defining its role in building cohesion while the system is running accordingly.

Let us go through our database as we are using xampp server version 127.0.0.1

1.1 SECTION 1:

1.1.1 ENTITIES

Our system has 8 entities including admin entity, that we are going to describe one by one according to their relationships;

A) CUSTOMER ENTITY

This table will be for the customer of the company, where information relates to the customer will be stored. The entity has its relationship with products entity because it will be necessary to know what kind of product does the customer purchased.

Tables
Attribute Description
CustID: Customer identity
ProID: Product identity
CFname: Customer first name
Clname: Customer last name
Cemail: Customer email
Cphone: Customer phone contact
Amountpaid: Amount paid by customer

B) EMPLOYEE ENTITY

This table will store data of managers for the company that will manage all assets in different branches. For this company has many branches in different parts of the country, it will be essential for CEO to coordinate data of each manager according to his/her responsibilities.

This entity has its relationship with the market entity because each market will have an employee in charge of controlling it. It also has a relationship with the property entity because the property will be controlled by managers, so we need to know the company's properties and the manager controlling them to ensure better utilization of them.

Tables
Attribute Description
EmpID: Employee identity
Fname: Employee first name
Lname: Employee last name
DOB: Employee date of birth
Email: Employee email
Contact: Contact
Department: The department of the employee
Salary: Employee salary
Contract: Contract signed by employee

C) FINANCE ENTITY

Finance entity will store financial data of the company. With this table, the company controls the financial circulation in the company by knowing how much is given to each employee per month and per year.

This entity has a relationship with the employee entity because finance will need to report how much goes out and to whom.

Tables
Attribute Description
FinID: Finance identity
branch_name: Branch name where employee works
asset_value: Asset value of the branch
salary_amount: Salary amount per month
amount_per_year: Amount per year
EmpID: Employee identity

D) MARKET ENTITY

In the market entity, the company can know the available market that has to be supplied in the future. Anyone that visits this table can get data related to the market.

The table has its relationship with the employee entity because each market needs the employee that will manage and control it. It also has a relationship with the product entity for products have to be specified by its type, that will be supplied in that market.

Tables
Attribute Description
MarkID: Market identity
EmpID: Employee identity
ProID: Product identity
Mprovince: Market province
Mdistrict: Market district
Msector: Market sector
Supplydate: Date to supply products

E) PRODUCT ENTITY

Product entity will store all data of products that the company produces by its type and raw materials needed to manufacture them.

It has a relationship with the raw material entity because we will need to know which resources products are manufactured from.

Tables
Attribute Description
ProID: Product identity number
RmID: Raw material identity number
Pname: Product name
Amount: Product amount
Price: Product price
Mnfdate: Manufactured date
Expdate: Expired date

F) PROPERTY ENTITY

As the company has assets, the table to manage those properties is needed. This table has a relationship with the employee entity to know who is controlling those properties.

Tables
Attribute Description
PropID: Property identity number
EmpID: Employee identity number
Pname: Property name
Province: The province of property
District: District where property is located
Sector: Sector of property

G) RAW MATERIAL ENTITY

In this entity, all data related to raw materials will be stored in it. Since it's a company for production, there will be raw materials used in the production process to get final products.

This table has a relationship with the supplier entity. He/she will be the supplier of raw materials that the company orders.

Tables
Attribute Description
RmID: Raw materials identity number
Rtype: Raw materials type
Ramount: Raw materials amount
stored_date: Its stored date
unstore_date: The date in which it's unstored
SupID: Supplier identity number

H) SUPPLIER ENTITY

This is the entity that will store information about the supplier. He/she is the supplier of raw materials that the company uses.

The table has a relationship with the raw materials table because the value and amount of raw materials need to be assigned to the corresponding supplier.

Tables
Attribute Description
SupID: Supplier identity number
Fname: Supplier first name
Lname: Supplier last name
Province: Province where supplier is located
District: District
Sector: Supplier’s sector
Phone: Contact of supplier
Email: Email of supplier
amount_paid: Amount received as payment

Chapter 1: Java Programming

1.Registration form

Here the user will register by filling this form with correct credentials as required by the system.

2.Login form

After the pop up of registration successfully message, the user need to click on OK then login form will directly displayed that will require the user to fill the form with correct credentials.

3. Employee form

After the pop up of registration successfully message, the user need to click on OK then login form will directly displayed that will require the user to fill the form with correct credentials.

4. Finance form

At this page the user will be having information stored in company database relates to financial circulation. The user can perform different actions includes insert, view, update and delete.

5. Product form

The product page show the list of products available in the stock of company. The user will be able to track information of products using product page that interact with database as he/she insert, view, update and delete. the user can achieve this when he fill text field as printed in product page

6. Properties form

The properties page will display all information of properties available in company database. The use will login and click at properties menu then the page the hit view button to check the properties of the company. He/she have more than one choice will navigating this page. He can create new information by filling the text field the insert button or update an existing information or delete expired information as well.

7. Raw material page

At this page, the user is able to check raw materials of the company. He/she can interact with company database at raw materials entity through use of buttons available at the page. He/she can insert, view, update and delete accordingly. Insert means adding new raw materials, view means checking the available raw materials entity, update means changing existing data and delete means removing data from database.

8. Supplier form

By here the user is able to track information relates to supplier of the company. He/she achieves this by using button on bottom row of the page that are insert, view, update and delete. If he/she insert, he is adding new supplier in database. If he/she view, he is trying to display supplier data stored in database and if he/she update means changing information of a supplier od he/she can delete an expired data.

9.Market page

The market page is for company’s market waiting to be supplied. Here the user will login and choose market menu then he/she can interact with database in market entity by using market page. The user will perform CRUD using buttons like insert, view, update, and delete.

10.Customer form

With customer form, the user will be able to track data of company’s customers. This will be done through insert, view, update and delete buttons.
